About the FBF-system

This is a short orverview of the FBF-system. For a more complete overview see Annex A in the Early Action Protocol.

{{ "1. Trigger Levels from Historical Analysis" | translate }}

For selected water-discharge measuring stations in Zambia, historical data on water-discharge levels (m3/s) was collected. This was combined with historical data on where and when floods were recorded in Zambia. This combination of data leads to two outcomes: (1) which levels of water discharge lead to floods, or what is the 'trigger-level'. These levels were established per station in close collaboratoin with the Trigger Working Group. (2) where do floods occur, when a certain station passes the trigger. This leads to a mapping of which districts are associated with which stations.

{{ "2. Daily incoming forecast" | translate }}

{{ "For all the stations where trigger-levels were established, now daily forecast data is coming in on predicted water discharge for 3 days from now and for 7 days from now. (These two lead-times are chosen, because they correspond to relevant timeframes to perform chosen Early Actions.) For each station, the forecasted water discharge is compared to the trigger-level. If it's exceeded, then a trigger is passed for that station. In turn, this means that a trigger is for the districts associated with that station." | translate }}

{{ "3. Establish Flood Extent" | translate }}

{{ "For each district where the trigger is passed, the exact forecasted water discharge is judged to determine the Return Period of the associated flood: the higher the value, the higher the Return Period. If a district has (e.g.) a predicted flood with Return Period 10 years, then for that district a portion of the national 10-year Return Period flood extent map is taken. This is done for all districts, after which all portions are pasted back together to form one flood extent map. In this map some districts might have a 10-year Return Period flood extent, some might have a 20-year Return Period flood extent (etc.) and some districts will have no flood. " | translate }}

{{ "4. Establish Exposure" | translate }}

The flood extent maps is combined with a population exposure grid. This way, the exact number of people living inside the forecasted flood extent can be counted. This forms the 'Exposed population'. The same procedure is followed for Cropland and Livestock. Other exposure-layers - such as roads, health sites and waterpoints - are available in the dashboard, for the user to visually compare with the flood extent.




{{ "5. Data visualized in dashboard" | translate }}

{{ "The outcomes of this daily run are uploaded to this dashboard (flood or no flood). This implies that most days of the year, this dashboard will reflect a no-flood scenario. See below for a tutorial on how to use the dashboard." | translate }}

{{ "6. Email notification system" | translate }}

{{ "If a trigger is passed, even for only 1 district, then an email notification is sent to the email-list. The email will contain basic information, and includes a link to this dashboard for more information. In the dashboard, new users can sign up to this emailing list." | translate }}
